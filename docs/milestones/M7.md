# M7.1 — Cognitive Reasoning Layer v1: Storage Foundations

## What was built
- Introduced a Postgres “mirror” table: `reasoning_chain_summaries`
- Defined Neo4j reasoning graph conventions (nodes + relationships) for future steps

## Why this was built
MIRA’s cognitive layer requires storing reasoning behavior. Neo4j will store the true reasoning graph,
while Postgres holds compact chain summaries for fast reporting, auditing, and reinforcement metrics.

## Neo4j (Primary Store) — Conventions
### Nodes
- `(:Concept {concept_id, ...})` — existing, stable concept root id
- `(:ReasoningChain {chain_id, query_text, created_at})`
- `(:ThoughtStep {step_index, note, score, created_at})`

### Relationships
- `(rc:ReasoningChain)-[:HAS_STEP]->(ts:ThoughtStep)`
- `(ts:ThoughtStep)-[:REFERS_TO]->(c:Concept)`
- Future (M7.3+):
  - `(c1:Concept)-[:DYNAMIC_LINK {weight, last_reinforced_at}]->(c2:Concept)`

## Postgres (Mirror Store)
### Table: reasoning_chain_summaries
Stores compact records for each chain:
- chain_id (uuid string)
- query_text
- root_concept_id (+ optional version)
- steps_summary (JSONB)
- outcome_success (bool/nullable)
- use_count, strength
- created_at, last_used_at
