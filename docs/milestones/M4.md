# Milestone M4 — Query Logging (Signals) in Postgres

## Goal
Log user queries as signals for future cognitive learning.

## What I Built
- Postgres SQLAlchemy engine + session
- QueryLog table (query_logs)
- POST /queries/log
- GET /queries/recent
- Integration test validating query log persistence

## Why It Matters
Query logs are the raw signals used later for:
- reinforcement
- gap detection
- drift detection
- quality scoring

## Checkpoint
- Queries can be logged and retrieved
- Tests pass and reports/logs are generated automatically
